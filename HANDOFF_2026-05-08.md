## Critical Operational Lessons

### Net new from 2026-05-08

**9. FRED's WAF slow-lanes browser User-Agents on the CSV endpoint.** Counterintuitive: most CDNs block bots and allow browsers. FRED is inverse — they expect humans on the chart page and programmatic clients on `/csv`. Browser UAs hitting CSV directly look like scrapers masquerading. Fix: omit custom User-Agent (let `requests` send default `python-requests/X.Y`). Encoded as comment in `terminal3c_claims_data.py`. **Audit any future puller before shipping a custom UA — don't ship one unless required by the API contract.**

**10. "Daemons are running" is a claim, not evidence.** The prior handoff said T3b's nowcast puller and BLS actuals were "running" / "scheduled." They had run exactly once, on April 26, and exited. No daemon. No plist. The trader silently rejected signals for 12 days. **Verification must come from log freshness, not from the handoff doc.** New protocol: every resume session, run a quick health check on data file mtimes for each engine before any other work.

**11. Live mode tests that dry-run hides.** T3c's `ShadowLedger.open()` was missing the `venue="kalshi"` arg — bug that only surfaced in live mode (dry-run skips the write). Five days of dry-run "validation" never touched it. **Always run a single `--once` cycle in live mode before flipping a daemon to live, and grep the ledger for the new row before walking away.**

**11b. Same pattern repeats — T1 NWS actuals went dark for 7 days while the trader kept opening positions.** Discovered 2026-05-08 17:20 UTC during resume health check. The settlement reconciler was looping hourly with `closed=0` since May 2 08:05 UTC. Pattern is now well-defined: if a puller can fail silently and a trader can keep firing without it, the next session will be eating into stuck capital. Fix shipped same hour: `portfolio_freshness_watchdog.py` + `com.portfolio.freshness-watchdog.plist` (15-min sweep over every data file in every engine). When stale → writes `~/Documents/freshness_alarm.flag`. **All traders must add a one-line check at the top of any open() codepath:**
```
if (Path.home() / "Documents" / "freshness_alarm.flag").exists():
    return  # refuse to open new positions while data is stale
```
Without this, the watchdog only alerts; with it, the watchdog blocks bad bets. Add it to every trader before the next decision-gate event.

**12. Empirical σ tables need n≥10 per bucket and a fallback flag.** T3b's `empirical_sigma.json` is exemplary — `min_n_per_bucket: 10`, lead-bucketed, `fallback: false` indicator, raw vs std documented. T3a/T3c should adopt the same shape if they ever build calibration tables.

### Carried forward from prior handoff (still binding)

1. **Reconciler timezone gate:** use `target_date + 32h UTC`. Don't relax.
2. **Kalshi market statuses:** both `settled` AND `finalized` are terminal. Always check both. Verified again on T3c reconciler 2026-05-08 — worked correctly.
3. **NWS actuals merge-on-date:** running daily aggregates require merge, not skip-if-date-exists.
4. **σ-too-tight defenses:** probability clip [0.05, 0.95], boundary filter, EV filter. Don't remove.
5. **Side-aware calibration math:** `_model_win_prob` returns `our_p` for YES, `1-our_p` for NO. Always use this wrapper.
6. **Real edge vs mid edge:** `edge = our_p - yes_ask` for YES, `edge = yes_bid - our_p` for NO.
7. **Daemon paths:** Mac processes run from `/Users/stevenemery/Documents`. Agent shell uses `/sessions/<name>/mnt/Documents`. Don't conflate.
8. **Picks pipeline: archetype-driven, not whitelist-driven.** Theses earn the right to fire from settled outcomes (n≥3).
