## HIGH (fix this week)

### T1

**H-T1-1. Bucket gate boundary mismatch — actual line of code**
`terminal1_phase2_paper_trader.py:468`
Gate is `entry < T1_ENTRY_MIN or entry > T1_ENTRY_MAX` — passes `entry == 0.50`. Validated bucket from `t1_edge_analysis.py:172` is half-open `[0.40, 0.50)` — markets at exactly 0.50 were classified into the next decile during the n=24 analysis. Trades at $0.50 leak into a non-validated regime. Fix: `>` → `>=`. One char.

**H-T1-2. Bankroll drop $5K → $1K is not enforced anywhere in code**
`terminal1_phase2_paper_trader.py:94, 619`
No `BANKROLL_USD` constant, no open-exposure check, no Kelly. `DEFAULT_CONTRACTS=10` × cluster cap × bucket count can exceed $1K. Either add an exposure check or shrink defaults. Cited "fee-aware Kelly" in audit prompt is **not implemented in T1**.

**H-T1-3. `already_open_on_ticker` doesn't honor `annul_close`**
`terminal1_phase2_paper_trader.py:321-343`
After an annul, function still returns False → duplicate open on same ticker possible. Mirror the cluster-cap path's annul handling.

**H-T1-4. Settlement reconciler defaults `threshold_above` for ambiguous T-strikes**
`terminal1_settlement_reconciler.py:149-154`
Silent assumption. Only fires when `signal_metadata.strike_display` is missing (legacy entries), but if it ever fires on a real `<N°` market it settles wrong. Fail-closed instead.

### T3a

**H-T3a-1. `kalshi_fee` rounding wrong (Kalshi uses ceiling, code uses round-to-cent + `+0.005`)**
`terminal3a_shadow_executor.py:182-184`
Causes EV-after-fee to clear `min-profit-usd` when reality wouldn't. Replace with `math.ceil(raw * 100) / 100`.

**H-T3a-2. `snapshot_event_legs` silently drops legs with no orderbook**
`terminal3a_shadow_executor.py:152-154`
If 1 of 5 legs returns None on a transient API error, `evaluate_arb` proceeds on a partial event; the missing leg may be the one that settles YES → total payout $0 instead of $contracts_per_leg. Fix: bail on partial.

**H-T3a-3. FOMC observer aggregates `sum_ask` even when legs have no quote**
`terminal3a_fomc_observer.py:167-174`
Substitutes 100¢ for missing legs → meaningless sums pollute the rolling baseline → fake dislocations from quote appearance/disappearance. Require `n_legs_with_ask == n_legs` before contributing to the baseline.

**H-T3a-4. Executor refuses arb when leg size < `contracts_per_leg` instead of resizing**
`terminal3a_shadow_executor.py:222`
`if avail_size < contracts_per_leg: return None`. A real arb on a small leg is rejected entirely. Resize to `min(yes_ask_size)` across legs.

**H-T3a-5. `state["processed"]` is permanently sticky**
`terminal3a_shadow_executor.py:400-405`
Skip-due-to-fees on first crossing → never re-evaluates subsequent crossings of the same flag combo. Key on `(event, flag, snapshot_id)` or expire skip entries.

### T3b

**H-T3b-1. Reconciler ignores Kalshi finalization state**
`terminal3b_settlement_reconciler.py:165`
Settles on BLS row presence alone. Never queries `expiration_value` / `result` / `status` from Kalshi (kalshi_logger captures these and they're never consulted). For paper-only this is "internally consistent." If you go live, this is a fail-closed bug.

**H-T3b-2. Annul-close ordering bug**
`terminal3b_settlement_reconciler.py:137-153`

Mirror the canonical `compute_state` from `shadow_pnl_core`. Low blast radius for May 13 (no annulments pending) but worth fixing.

**H-T3b-3. Paper trader has dead seed-loop**
`terminal3b_paper_trader.py:497-500`
`for s in accepted: cluster_seen[...] = ...; break` is a no-op overwritten by the proper loop right below. Cosmetic but confusing.

### T3c

**C-T3c-1 (Critical for May 14 idempotency). Reconciler is NOT idempotent on double-fire**
`terminal3c_settlement_reconciler.py:204-210`
`shadow_pnl_core.ShadowLedger.close()` raises `ValueError` if position is already closed; outer `except Exception` catches but aborts the rest of `reconcile_once` → other positions skipped, per-event rollup never writes. If the Cowork task fires twice, second run errors and contaminates the log. Wrap in `try/except ValueError`.

**H-T3c-1. ICSA prints 05:30 PT, reconciler runs 07:30 PT — single-shot with no retry**
Kalshi settlement timing varies (minutes to hours). `--once` exits 0 with `pending=1` if Kalshi hasn't settled by 07:30. No alert. Add a second `--once` fire later in the day OR a checklist item to re-run if `pending=1`.

**H-T3c-2. `find_open_t3c_positions` has same annul-close ordering bug**
`terminal3c_settlement_reconciler.py:85-100` and `terminal3c_paper_trader.py:253-269`
Same fix as H-T3b-2. Use `compute_state` from `shadow_pnl_core`.

**H-T3c-3. Stale docstring claim — MIN_EDGE_YES is $0.10 not $0.15**
`terminal3c_paper_trader.py`
Docstring says $0.15, code uses $0.10 (post 2026-05-03 real-edge math fix). Align docs to code.
